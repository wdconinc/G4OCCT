# G4OCCT — Geometry Test Status (Milestone v0.5 Tracker)

This document tracks the current validation state of every STEP fixture in the
geometry test suite and records the outstanding work required to reach
**Milestone v0.5** (full test suite passing for all G4 primitives).

Fixtures are grouped by family.  Each fixture has a STEP file plus a
`provenance.yaml` describing its construction parameters.  The current test
suite checks OCCT volume computations against expected values; the planned
extension adds full ray-comparison checks that verify `Inside`,
`DistanceToIn(p, v)`, and `DistanceToOut(p, v)` against native Geant4 results.

### 3.1 Faceted approximation (`G4Hype`, `G4Paraboloid`)

Both solids have analytic curved surfaces (hyperboloid sheet, paraboloid of
revolution) that OCCT cannot represent exactly as standard NURBS surfaces.
The STEP fixtures approximate the curved boundary by lofting through a
discretised set of profile curves.  This faceting causes systematic
differences in `DistanceToIn`/`DistanceToOut` near curved faces.

**Work required:**
1. Evaluate whether a high-degree NURBS approximation within `≤ kCarTolerance/10`
   is feasible using OCCT's `GeomConvert` utilities.
2. If so, regenerate fixtures with the tighter-tolerance surface representation
   and enable full ray-comparison checks.
3. Alternatively, implement a Geant4-side fallback so that `G4OCCTSolid`
   delegates navigation to the analytic Geant4 solid for these specific classes.

### 3.2 Frame alignment (`G4Ellipsoid`, `G4EllipticalCone`, `G4EllipticalTube`, `G4Polycone`, `G4Polyhedra`)

Profile-loft construction can place the STEP origin at a coordinate that
differs from the Geant4 local-frame convention.  See
[Reference Position Handling](reference_position.md) for recentering strategies.

**Work required:** For each class, verify the Geant4 local-frame origin
convention, adjust the fixture generation script to recenter the STEP solid,
and update `provenance.yaml` to record the centering.

| `G4VTwistedFaceted` | `faceted-dz20-theta8-phi20-v1` | ✅ | Ruled-loft surrogate |

### 4.1 Ruled-loft surrogates for twisted solids

The current STEP fixtures for `G4TwistedBox`, `G4TwistedTrap`, `G4TwistedTrd`,
`G4TwistedTubs`, and `G4VTwistedFaceted` are generated by a ruled-loft
utility that approximates the twisted shape with a sequence of ruled surface
patches.  The resulting BRep differs geometrically from the analytic Geant4
solid, so ray-intersection distances and inside/outside classifications diverge
near the boundary.

**Work required:**
1. Implement a Geant4-to-OCCT direct construction path for each twisted class
   (parameterise the swept profile at discrete twist angles and build a proper
   B-spline surface through those sections).
2. Regenerate the STEP fixtures so that the STEP boundary matches the Geant4
   analytic solid to within `kCarTolerance`.
3. Update `provenance.yaml` files to record the new construction method and
   confirm CI passes with full ray-comparison checks enabled.

## 9. Adding or Updating Fixtures

To **add** a new fixture:
1. Generate a STEP file using the appropriate DRAWEXE or OCCT utility script
   in `src/tests/fixtures/geometry/<family>/`.
2. Write a `provenance.yaml` describing the construction parameters.
3. Register the fixture in the family `manifest.yaml` with
   `validation_state: validated` and a volume expectation.
4. Run the geometry validation test and confirm no errors are reported.

To **update** an existing fixture (e.g., to fix frame alignment):
1. Regenerate the STEP file using the updated construction script.
2. Normalise the STEP FILE_NAME timestamp with
   `src/tests/fixtures/geometry/tools/normalize_step_header.py`.
3. Update `provenance.yaml` to describe the change.
4. Confirm the volume expectation still holds and the validation test passes.

See [Reference Position Handling](reference_position.md) for recentering
strategies and [Solid Navigation Design](solid_navigation.md) for tolerance
and unit-convention requirements that ray-comparison checks must satisfy.
